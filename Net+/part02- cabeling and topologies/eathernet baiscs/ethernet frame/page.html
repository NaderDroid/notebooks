the frame of eathernet consists of many parts and here to review them:
a- preamble: this is a noisy frame to inform the NIC that there is a frame that is coming
b- destination mac address
c- source mac address
d- data type = ether type
e- data: this has minimum of 64 bytes and 1522 bytes max. if less than 64 then a 'padding frame will be used to suffice the size' if more than 1522 bytes then the frame will be split and sequence to manage the seperation process.
f- frame check sequence FCS: to check the correctness of the frame
