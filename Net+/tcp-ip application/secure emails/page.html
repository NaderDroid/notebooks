Confusing topic but i will try to sum up the important points here

1- email services was not intended to be secure from old days
2- people started to think about securing their emails
3- TLS was added as a layer to the traditional smtp pop3 and imap protocols
4- this was not very effeicint for different reasons the main reason is that it used to start the connection in unencrypted state and then switch to the encrypted designated port. This could be easly broken with man in the middle attack and we started to think about different ways
5- StartTLS comes in and combines all ports into the port number 465 that was changed later because it used to confuse the mail server as it was supporting the same port for both TLS and StartTLS
6- StartTLS moved to port 587 as its default port on the tcp protocol

=== review

- smtp pop3 and imap are unencrypted

- the STARTTLS extension uses only port 587 for encrypted communication
