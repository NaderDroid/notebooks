This is heavy topic and here I am going to summerize the lecture

- there are two dns software in use today 1- bind 2- microsoft dns server

- interior dns is used to connect hosts in your home or company and usually ends with local, home or lan tld

- the term 'authoritative dns' means that this server is able to resolve FQDN to ip address on that TLD

- the resolution process can be done with the help of whats known as 'lookup zones' records that holds information of different hosts on that particular dns server

- dns can be redundent to increase availability however there must be one SOA (start of autority) for each TLD

- CName (Canonical name) is an alias that we make to point to a particular host or ip

- when sending mails the mail client resolves the ip address of the mail server using the MX record (mail exchange) on the dns server to accuire and deliver the mail to its destination

- reverse lookup zone is used to resolve ip address to FQDN and its used to reduce spams on mail servers

- SRV records are used mainly for services like voip and sip protocol

- TXT record was used for clarification purposes however nowadays its very important especially in the anti-spam world

- CName are used to make alias of hosts on the dns
